What are Data Types in Python?

Data types in Python are classifications that specify the type of value a variable can hold. Python is a dynamically typed language, which means you don't need to explicitly declare the data type of a variable when you create it. The Python interpreter automatically determines the data type based on the value assigned to the variable.

Key Features of Python Data Types

  • Dynamically typed - no need to declare types explicitly
  • Type checking happens at runtime
  • Variables can change types during execution
  • Built-in type() function to check data types
# Python automatically determines data types
x = 10          # int
y = 3.14        # float
name = "Python" # str
is_true = True  # bool

# Check data types
print(type(x))     # 
print(type(y))     # 
print(type(name))  # 
print(type(is_true)) #

Numeric Types

1. Integer (int)

Integers are whole numbers without decimal points. Python 3 has unlimited precision for integers.

# Integer examples
positive_num = 42
negative_num = -17
zero = 0
large_num = 999999999999999999999

# Binary, octal, and hexadecimal
binary = 0b1010      # Binary (equals 10 in decimal)
octal = 0o755        # Octal (equals 493 in decimal)
hexadecimal = 0xFF   # Hexadecimal (equals 255 in decimal)

print(type(positive_num))  #

2. Float

Floating-point numbers represent real numbers with decimal points.

# Float examples
pi = 3.14159
temperature = -273.15
scientific = 1.5e10  # Scientific notation (15,000,000,000)

# Special float values
infinity = float('inf')
negative_infinity = float('-inf')
not_a_number = float('nan')

print(type(pi))  #

3. Complex

Complex numbers have real and imaginary parts.

# Complex number examples
z1 = 3 + 4j
z2 = complex(2, -5)  # 2 - 5j

# Accessing real and imaginary parts
print(z1.real)  # 3.0
print(z1.imag)  # 4.0

# Complex operations
result = z1 + z2
print(result)    # (5-1j)

Text Type - String (str)

Strings are sequences of characters enclosed in quotes. Python supports single, double, and triple quotes.

# String creation
single_quote = 'Hello World'
double_quote = "Python Programming"
triple_quote = """This is a
multi-line
string"""

# String operations
name = "Alice"
greeting = f"Hello, {name}!"  # f-string formatting
concatenated = "Hello" + " " + "World"
repeated = "Ha" * 3  # "HaHaHa"

# String methods
text = "  Python Programming  "
print(text.strip())      # Remove whitespace
print(text.upper())      # Convert to uppercase
print(text.lower())      # Convert to lowercase
print(text.replace("Python", "Java"))  # Replace substring
Method Description Example
len() Returns string length len("Hello") → 5
str.split() Splits string into list "a,b,c".split(",") → ["a", "b", "c"]
str.join() Joins list into string ",".join(["a", "b"]) → "a,b"
str.find() Finds substring index "hello".find("ll") → 2

Sequence Types

1. List

Lists are ordered, mutable sequences that can store different data types.

# List creation and operations
fruits = ["apple", "banana", "orange"]
numbers = [1, 2, 3, 4, 5]
mixed = [1, "hello", 3.14, True]

# List methods
fruits.append("grape")        # Add element
fruits.insert(1, "kiwi")     # Insert at index
fruits.remove("banana")      # Remove element
popped = fruits.pop()        # Remove and return last element

# List comprehension
squares = [x**2 for x in range(10)]
even_numbers = [x for x in range(20) if x % 2 == 0]

print(fruits)      # ['apple', 'kiwi', 'orange', 'grape']
print(squares)     # [0, 1, 4, 9, 16, 25, 36, 49, 64, 81]

2. Tuple

Tuples are ordered, immutable sequences. Once created, you cannot change their elements.

# Tuple creation
coordinates = (10, 20)
colors = ("red", "green", "blue")
single_element = (42,)  # Note the comma for single element

# Tuple unpacking
x, y = coordinates
first, *rest = colors  # first = "red", rest = ["green", "blue"]

# Named tuples (requires import)
from collections import namedtuple
Point = namedtuple('Point', ['x', 'y'])
p = Point(10, 20)
print(p.x, p.y)  # 10 20

3. Range

Range objects represent immutable sequences of numbers, commonly used in for loops.

# Range examples
numbers = range(10)        # 0 to 9
even_nums = range(0, 20, 2)  # 0, 2, 4, ..., 18
countdown = range(10, 0, -1)  # 10, 9, 8, ..., 1

# Converting range to list
list_nums = list(range(5))  # [0, 1, 2, 3, 4]

# Using range in loops
for i in range(5):
    print(i)  # Prints 0, 1, 2, 3, 4

Mapping Type - Dictionary (dict)

Dictionaries are unordered collections of key-value pairs. Keys must be unique and immutable.

# Dictionary creation
student = {
    "name": "Alice",
    "age": 20,
    "grade": "A",
    "subjects": ["Math", "Physics", "Chemistry"]
}

# Dictionary operations
student["age"] = 21                    # Update value
student["email"] = "alice@email.com"   # Add new key-value pair
del student["grade"]                   # Delete key-value pair

# Dictionary methods
keys = student.keys()      # Get all keys
values = student.values()  # Get all values
items = student.items()    # Get key-value pairs

# Dictionary comprehension
squared = {x: x**2 for x in range(5)}  # {0: 0, 1: 1, 2: 4, 3: 9, 4: 16}

# Accessing with get() method (safe)
grade = student.get("grade", "Not Available")  # Returns default if key not found

Dictionary Best Practices

  • Use meaningful key names
  • Prefer get() method over direct access for optional keys
  • Use dictionary comprehensions for simple transformations
  • Remember that dictionaries maintain insertion order (Python 3.7+)

Set Types

1. Set

Sets are unordered collections of unique elements. They're useful for removing duplicates and set operations.

# Set creation
fruits = {"apple", "banana", "orange"}
numbers = set([1, 2, 3, 4, 5])
unique_chars = set("hello")  # {'h', 'e', 'l', 'o'}

# Set operations
fruits.add("grape")           # Add element
fruits.remove("banana")       # Remove element (raises error if not found)
fruits.discard("kiwi")        # Remove element (no error if not found)

# Set operations (mathematical)
set1 = {1, 2, 3, 4}
set2 = {3, 4, 5, 6}

union = set1 | set2           # {1, 2, 3, 4, 5, 6}
intersection = set1 & set2    # {3, 4}
difference = set1 - set2      # {1, 2}
symmetric_diff = set1 ^ set2  # {1, 2, 5, 6}

2. Frozenset

Frozensets are immutable versions of sets. They can be used as dictionary keys.

# Frozenset creation
immutable_set = frozenset([1, 2, 3, 4])
frozen_from_set = frozenset({1, 2, 3})

# Frozensets can be dictionary keys
cache = {
    frozenset([1, 2]): "result1",
    frozenset([3, 4]): "result2"
}

# All set operations work with frozensets
fs1 = frozenset([1, 2, 3])
fs2 = frozenset([2, 3, 4])
union = fs1 | fs2  # frozenset({1, 2, 3, 4})

Boolean Type (bool)

Boolean values represent truth values: True or False. They're subclasses of integers.

# Boolean values
is_student = True
is_graduated = False

# Boolean operations
result1 = True and False   # False
result2 = True or False    # True
result3 = not True         # False

# Comparison operations return booleans
age = 25
is_adult = age >= 18       # True
is_teenager = 13 <= age <= 19  # False

# Truthiness of other types
bool(1)          # True
bool(0)          # False
bool("hello")    # True
bool("")         # False
bool([1, 2])     # True
bool([])         # False
bool(None)       # False
Value Boolean Equivalent Explanation
0, 0.0 False Numeric zero values
"", [], {}, () False Empty collections
None False None value
All other values True Non-empty, non-zero values

Practical Examples

Type Conversion Examples

# Type conversion functions
# String to number
age_str = "25"
age_int = int(age_str)      # 25
price_str = "29.99"
price_float = float(price_str)  # 29.99

# Number to string
count = 42
count_str = str(count)      # "42"

# List to string and vice versa
words = ["hello", "world"]
sentence = " ".join(words)  # "hello world"
word_list = sentence.split()  # ["hello", "world"]

# Checking types
def process_data(data):
    if isinstance(data, int):
        return data * 2
    elif isinstance(data, str):
        return data.upper()
    elif isinstance(data, list):
        return len(data)
    else:
        return "Unknown type"

Real-World Example: Student Management System

# Student management system using different data types
students = [
    {
        "id": 1,
        "name": "Alice Johnson",
        "age": 20,
        "grades": [95, 87, 92, 88],
        "subjects": {"Math", "Physics", "Chemistry"},
        "is_enrolled": True,
        "contact": ("alice@email.com", "123-456-7890")
    },
    {
        "id": 2,
        "name": "Bob Smith",
        "age": 19,
        "grades": [78, 85, 80, 92],
        "subjects": {"History", "English", "Art"},
        "is_enrolled": True,
        "contact": ("bob@email.com", "987-654-3210")
    }
]

# Calculate average grade for a student
def calculate_average(grades):
    return sum(grades) / len(grades) if grades else 0

# Find students by subject
def find_students_by_subject(students, subject):
    return [student["name"] for student in students 
            if subject in student["subjects"]]

# Process student data
for student in students:
    avg_grade = calculate_average(student["grades"])
    print(f"{student['name']}: Average Grade = {avg_grade:.2f}")

math_students = find_students_by_subject(students, "Math")
print(f"Math students: {math_students}")

Memory and Performance Tips

  • Use tuples for data that won't change (coordinates, RGB values)
  • Use sets for membership testing (O(1) average time complexity)
  • Use dictionaries for key-based lookups
  • Lists are great for ordered data that needs modification
  • Consider using generators for large datasets to save memory

Summary

Python's built-in data types provide a solid foundation for programming. Understanding when and how to use each type effectively is crucial for writing efficient and readable code.

When to Use Lists

  • Ordered data that can change
  • Need to append/remove items
  • Duplicate values are allowed

When to Use Tuples

  • Immutable ordered data
  • Function return values
  • Dictionary keys (if hashable)

When to Use Dictionaries

  • Key-value relationships
  • Fast lookups by key
  • Configuration settings

When to Use Sets

  • Unique elements only
  • Set operations (union, intersection)
  • Fast membership testing
